Desenvolvi um framework de automação front-end low code utilizando Java e Selenium, criado para resolver uma dor interna da empresa: a dificuldade de alguns QA’s em programar e automatizar testes. O framework foi projetado para abstrair ao máximo as complexidades técnicas — como configurações de WebDriver, integrações com pipelines CI/CD e estrutura de código — oferecendo um projeto base (sample) pronto para uso, bastando o clone do repositório para iniciar a automação.

Todo o código segue os padrões de qualidade e segurança da Alelo, sendo continuamente monitorado e metrificado via SonarQube.

Sou responsável pela gestão e integração da device farm utilizada nos testes automatizados, garantindo sua disponibilidade e evolução contínua.

Promovo workshops internos para capacitação dos QA’s em automação e boas práticas de qualidade. Participo também da elaboração de frameworks back-end e da documentação de práticas de qualidade de software, disseminando padrões e acelerando a maturidade do time.
